## Surprises / deviations from the plan

- **The live full-agent run is impractical on the shared box** (qwen3:4b ≈ 140 s per
  decision — same constraint documented in M5.1). So skeleton v4 is proven in two honest
  layers: the deterministic variant (controllable fake LLM in the two judgment slots)
  proves the **wiring** of real-graphs → real-chain → real-controller in CI; the opt-in
  `A2A_LIVE_LLM=1` variant drives the same graphs with **real judgment** and is ready for
  a capable box, asserting valid behavior on whichever way the LLM decides.
- `ChainConsumerTools` gained an injectable httpx client so the same tool code drives a
  live controller or an in-process one (FastAPI `TestClient`) — the tool can't tell, which
  is exactly the point of the port seam.
