## Explain-back answers

1. **What does OZ's ERC721 give you that you didn't write?** The entire ownership ledger:
   `ownerOf(id)`, `transferFrom`/`safeTransferFrom`, approvals, and the safe bookkeeping
   behind them — the audited "who owns each unique token" half. We add only the terms (the
   `Entitlement` struct); ownership, where a bug means stolen tickets, is inherited not
   hand-rolled. (Landed via the "two ledgers" framing: struct = terms, ERC-721 = owner.)
2. **Why a harness instead of a public `mintForTest`?** A public mint would ship a permanent
   free-mint backdoor to the real chain — exactly the hole invariant I1 forbids. The harness
   is a test-only subclass exposing the `internal` `_issue` *within the test build only*, so
   tests can create tickets while the deployed contract keeps no public mint path.
