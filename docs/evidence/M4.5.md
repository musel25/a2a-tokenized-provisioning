## Surprises / deviations from the plan

- **Paid the M1.5 deferral:** `ChainReader` — `ChainClient` with the write half
  amputated (no `Account`, no signing middleware, no `sign_*`/`fulfill`). It reuses the
  read+watch code by *borrowing the unbound methods*, so there is no path — not even an
  unused one — from a controller to a private key. It satisfies `EntitlementReader`;
  `build_runtime` uses only it (rule 2's spirit, enforced by type).
- **Two autonomous drivers, both re-check the chain:** the Revoked watcher calls
  `service.handle_revoked` (which re-reads `.revoked` before acting — never trust the
  event), and `ExpiryTimer` polls `service.tick` (which re-reads chain time — ADR-004
  in code, a dumb poll because chain time can jump).
- The activation flow needed chain time advanced into the window first (the seeded
  chain starts at 13:32, before Ada's 14:00) — the test warps 1800 s, which is also
  exactly what a real consumer waiting for their window would experience.
