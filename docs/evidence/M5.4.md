## Surprises / deviations from the plan

- **In-process tool callables, not a stdio round-trip, for the same-process agents.**
  `chain_tools(client)` returns the five §6.1 operations as plain functions over one
  agent's `ChainClient`; the FastMCP `build_chain_mcp(client)` wraps those same functions
  for *cross-process* agents (M5.5+). The custody rule is about WHERE the key lives (in
  this agent's chainmcp instance), not the transport — so the in-process path honors it
  identically, and the tests don't need a subprocess dance.
- `ctrl-mcp` is realized as the consumer tool's three HTTP calls to the controller API
  (challenge → sign → submit — docs/03 §6.2's deliberate three), not a separate FastMCP
  process: the controller already speaks HTTP (M4.4), so the "MCP wrapper" is a thin
  client. The FastMCP server shell exists for when an out-of-process agent needs it.
