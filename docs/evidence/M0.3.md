## Explain-back answers (write them, two sentences each is enough)

> Understanding for this slice was built through the three-level explanation, the
> walkthrough doc [`docs/03c-skeleton-walkthrough.md`](../03c-skeleton-walkthrough.md),
> and the hands-on notebook [`e2e/notebooks/skeleton_explore.ipynb`](../../e2e/notebooks/skeleton_explore.ipynb)
> (build + inspect every component) rather than a separate verbal recap.

1. Why is it a *feature* that FakeChain is ~30 lines?
   > Because the skeleton exists to prove the architecture's *joints*, not to reimplement
   > Ethereum — a fake that is a dict, two balances, and a salt set has almost no behaviour
   > of its own to be wrong about, so a red test points at a wiring problem, not the fake.
   > The moment a fake grows real logic (gas, signatures, ordering) it becomes a second
   > implementation that can silently disagree with the real one and make tests lie; staying
   > tiny keeps it honest and keeps the FakeChain → chainmcp swap clean.
2. Which future phase replaces which fake — name all four swaps.
   > `FakeChain` → `chainmcp` (real chain client + signing) at **M1.5**; `FakeNet` →
   > `netctl` (`GnmiProvisioner`, real gNMI) at **M3.4** (lib at M3.2); `StubController` →
   > the real `controller` (predicate/domain + HTTP API + revoke watcher) at **M4.x**
   > (skeleton v3 at M4.5); `scripted_agents` → LLM agents (LangGraph + MCP + A2A) at
   > **M5.6**. Each fake fits a Protocol *port*, so every swap leaves the wiring unchanged.

## Surprises / deviations from the plan

- **Skeleton path.** Plan writes `e2e/skeleton/`; built at `e2e/src/e2e/skeleton/` so
  it is part of the installed `e2e` package and imports as `e2e.skeleton` (the package
  uses a `src/` layout). File names match the plan: `fakes.py`, `stub_controller.py`,
  `scripted_agents.py`.
- **Two teardown paths, on purpose (foreshadows M4.5).** Expiry is time-driven via
  `controller.tick()` (re-reads `chain_time()` each call, ADR-004); revocation is
  event-driven via the chain's `watch_revoked` callback. Each lifecycle test exercises
  one path. A single polling `tick()` that also checked `revoked` would be ~5 lines
  shorter but would leave `watch_revoked` dead and not rehearse the real watcher.
- **Signature is a valid placeholder, not literal `"0xFAKE"`.** `SignedOffer` validates
  the 65-byte hex pattern (M0.2), so the scripted provider returns
  `CANONICAL_SIGNED_OFFER` (placeholder sig `0xab…`). Fakes do not verify signatures
  anyway — that is the contract's job (M1.3) and the cross-stack test's (M1.5).
- **Entitlement id seeded to #7.** The real contract increments ids from 1; the fake
  starts its counter at `TICKET_ID` so the canonical lifecycle yields ticket #7 and the
  narration matches the story. Documented at the `next_id` parameter.
- **ABI params decode is two `int()` calls.** `FakeChain` slices the on-chain `params`
  blob (`0x` + capacity word + qos word) back into `BandwidthParams`. This is the fake's
  stand-in for chainmcp's real ABI decoder (rule 2); bandwidth (serviceType 0) only.
- **Resource map is in-memory.** The stub controller holds `_RESOURCE_MAP`
  (canonical `resource_id → RESOLVED_PATH`) as a stand-in for
  `controller/resource_map.yaml` (rule 6 / ADR-005); the real resolver arrives at M4.3.
- **Removed `e2e/tests/test_stage_exists.py`.** The M0.1 no-op smoke test; its own
  docstring said it is "replaced in spirit by the lifecycle tests at M0.3."
- **Tests beyond the required three.** `test_fakes_satisfy_ports` enforces rule 7
  (mocks satisfy the same Protocol via `isinstance`); `test_predicate` covers accept +
  six deny paths; `test_activation_denied_when_provisioner_fails` covers the
  netctl-said-no joint (apply fails → `E_NETWORK`, no ACTIVE session). Exhaustive
  predicate deny-path coverage and the full `FAILED` session state are M4.1/M4.5's job;
  this is an honest first pass.
- **Teaching artifacts shipped with the slice.** A code-level walkthrough
  (`docs/03c-skeleton-walkthrough.md`, with a backlink added from `03b`) and an
  interactive Jupyter notebook (`e2e/notebooks/skeleton_explore.ipynb`) that builds and
  inspects every component by hand. The notebook is run via the `demo` dependency group
  (`ipykernel`, `nbconvert`) and is excluded from ruff; CI never installs the group.
