## Surprises / deviations from the plan

- **The forge artifact path is source-file-keyed, not contract-keyed** —
  `out/Settlement.sol/A2ASettlement.json`. First loader assumed
  `A2ASettlement.sol/`; every chain test silently SKIPPED (the skip guard hid the bug)
  until the glob fix. Skip guards need their own sanity check.
- **`vm.prank`-style trap, uv edition:** the editable install of `chainmcp` was built
  while `src/chainmcp/` was still empty, so `import chainmcp` failed even though
  `uv pip list` showed it installed. `uv sync --reinstall-package chainmcp` fixed it.
- **Ticket #7 required making the story physical:** FakeChain is seeded `next_id=7`; a
  real chain starts at 1. Instead of weakening the assert, `seed_chain()` sells tickets
  #1–#6 (Bell → Carol, six salts) before every test run — and the balance asserts became
  *deltas*, because Bell now legitimately starts with 60 TOK of prior earnings.
- **The M1.3 error-name parity paid off unprompted:** `ChainRevert` carries the decoded
  custom-error name, and the e2e world maps the three shared names straight onto the
  fake's exception classes — the deny-path tests didn't change *at all* between
  profiles. `InsufficientFunds` is the one seam: on-chain it's the token's
  `ERC20InsufficientBalance/Allowance`, mapped explicitly.
- **CI's python job now installs Foundry** (`forge build` before pytest) — otherwise the
  cross-stack tests skip in CI and the most failure-prone seam goes unwatched. Mirrored
  in docs/05 step 7.
- **The adversarial review (58-agent workflow, 2 rounds — the first died on a session
  usage limit) confirmed 25 findings; all fixed pre-merge.** The heavyweight ones:
  the `watch_revoked` polling thread could **die silently on any RPC hiccup or callback
  exception** — for the M4.5 controller that's a revoked entitlement keeping its live
  session, a teardown path failing *open* (fixed: per-tick and per-callback isolation,
  at-least-once delivery, `last_watch_error` observability, baseline block read
  synchronously at registration, watch-after-close raises); Mallory's key literal had
  crept into e2e (rule 2 — moved into `chainmcp.testing.ANVIL_KEYS`); `get()` decoded
  any serviceType ≥ 2 as telemetry and crashed in eth-abi instead of refusing by name;
  a refused fulfill left a standing ERC-20 allowance (now withdrawn best-effort, so
  "left no trace" is true on-chain too); `launch_anvil`/seeding could orphan the anvil
  process on deploy failure. Deferred with a dated note: a keyless read-only ChainClient
  for the controller (M4.5).
