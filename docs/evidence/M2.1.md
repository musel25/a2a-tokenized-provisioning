## Surprises / deviations from the plan

- **`startup-config:` is broken under sudoless containerlab** — the overlay postdeploy
  task dies (`/tmp/clab-overlay-config: No such file or directory`) and the router boots
  with every port admin-down, *silently*. First ping simply timed out; `show interface
  brief` was the tell. Fix: bind-mount `srl1-init.cli` + an `exec` that replays it
  through `sr_cli` after boot — deploy is one-shot again (verified by destroy+redeploy).
- **A zombie lab was squatting the RAM budget**: `clab-bandwidth-poc-*` containers from a
  *deleted* repo (`~/Github/srl-gnmi-bandwidth-poc` no longer exists; its 3 SR Linux
  nodes exited 2 months ago, 4 host containers still running). Removed the leftover
  containers with `docker rm -f` — SR Linux needs ~2 GB and the machine has 14 total.
  (The unrelated `partyroom-postgres-dev` container was left untouched.)
- sr_cli one-liner chaining with `;` doesn't parse in a single `-e` string; feeding the
  command file via stdin (`docker exec -i … sr_cli < file`) is the reliable non-interactive
  path, wrapped by `enter candidate` / `commit now`.
- Plan named `ghcr.io/srl-labs/network-multitool`; the image cached on this machine is
  its predecessor `ghcr.io/hellt/network-multitool` (same tool, has iperf3) — used as-is
  to avoid a pull.
