## Surprises / deviations from the plan

- **Stack too deep, and a false fix-comment caught by review.** `hashOffer`'s 13-word
  `abi.encode` fails to compile on unoptimized legacy codegen. First fix enabled
  `via_ir` with a comment claiming it was required; the adversarial review *proved the
  comment false* (`optimizer = true` alone suffices) — and flagged that via-ir degrades
  forge trace source maps, which this repo values. Final: optimizer only, honest comment.
- **`vm.prank` eaten by a hidden external call.** `settlement.fulfill(offer, _sign(offer))`
  fails mysteriously: `_sign` calls `settlement.hashOffer` while evaluating arguments,
  consuming the prank, so `fulfill` ran as the test contract. Sign first, then prank.
- **Malformed (garbage) signatures revert with OZ's `ECDSAInvalidSignature*`, not our
  `BadSignature`** — only well-formed signatures by the wrong signer produce
  `BadSignature`. Both refuse, so no invariant is harmed; noted for M1.5's error mapping.
- **ERC-20 and ERC-721 `Transfer` events share the same topic0** — Ada's fulfill tx shows
  two identical-looking `Transfer` topics; you tell money from ticket by emitting address.
- **`cast wallet verify --data --from-file`** validates an EIP-712 signature locally —
  an unplanned, chain-free cross-check that foreshadows the M1.5 cross-stack test.
- **Fixture times are now in the past** relative to wall clock; the lab pins Anvil to
  story time with `--timestamp 1757944500` — accidentally a perfect ADR-004 demo.
